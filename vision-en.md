# ⚙️ Role-Driven Panel Architecture

One of the strongest ideas in Zainlak is that its dashboards are **not duplicated admin apps**.

They are generated through a **configuration-driven panel engine**.

## Core Idea

Zainlak’s dashboard system is built as **one frontend project** capable of producing multiple isolated panels through a **JSON-spec-driven architecture**.

Powered by:

- **Next.js**
- **Redux**
- **Radix UI**
- **shadcn/ui**

Panels can be generated by runtime role such as:

- `ROLE=admin`
- `ROLE=vendor`
- `ROLE=delivery`
- `ROLE=affiliate`

Each panel is built according to role, not merely hidden behind conditional UI logic.

## Why This Matters

- stronger role isolation
- better security posture
- cleaner architecture
- lower duplication
- consistent design language
- easier panel expansion
- each dashboard behaves like its own operational product

This is one of Zainlak’s biggest strategic advantages.
